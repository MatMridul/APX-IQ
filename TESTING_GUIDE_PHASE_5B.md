### Common Issues and Solutions

#### Issue 1: Backend Won't Start
**Error**: `Address already in use`

**Solution**:
```bash
# Find process using port 8000
lsof -i :8000  # Mac/Linux
netstat -ano | findstr :8000  # Windows

# Kill the process
kill -9 <PID>  # Mac/Linux
taskkill /PID <PID> /F  # Windows
```

#### Issue 2: No Laps in Dropdown
**Symptoms**: Dropdown shows "No recorded laps found"

**Solutions**:
1. Verify backend is running: `curl http://localhost:8000/health`
2. Run populate script: `python scripts/populate_test_laps.py`
3. Check backend logs for errors
4. Click "🔄 Refresh Laps" button

#### Issue 3: Report Generation Fails
**Error**: "Failed to generate report"

**Solutions**:
1. Check backend logs for detailed error
2. Verify intelligence layer is online: `curl http://localhost:8000/intelligence/health`
3. Try with mock data enabled
4. Check if Ollama is running (if using Ollama backend)

#### Issue 4: Report Takes Too Long
**Symptoms**: Loading for >60 seconds

**Solutions**:
1. Check which backend is active (Ollama is slowest)
2. Set up Gemini API key for faster generation:
   ```bash
   export GEMINI_API_KEY="your-key-here"
   ```
3. Restart backend to pick up new API key
4. Template mode is instant but less detailed

#### Issue 5: Frontend Build Errors
**Error**: TypeScript compilation errors

**Solutions**:
```bash
cd ui
rm -rf .next node_modules
npm install
npm run dev
```
