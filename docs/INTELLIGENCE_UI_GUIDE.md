# APX IQ Intelligence Dashboard - User Guide

## Overview

The Intelligence Dashboard is your AI-powered race engineer, providing detailed lap analysis and coaching insights using advanced telemetry processing and natural language generation.

## Accessing the Intelligence Dashboard

### From the Main Dashboard

1. Start your race session and navigate to the main dashboard at `http://localhost:3000/dashboard`
2. Look for the **"INTELLIGENCE"** button in the top-right header (gold accent, brain icon)
3. Click the button to navigate to the Intelligence Dashboard

### Direct Access

Navigate directly to: `http://localhost:3000/dashboard/intelligence`

## Dashboard Features

### 1. Backend Status Panel

**Purpose**: Verify that the intelligence layer is online and check which LLM backend is active.

**How to Use**:
- Click the **"CHECK STATUS"** button
- The panel will display:
  - ✅ **Status**: Online/Offline
  - 🤖 **LLM Backend**: Ollama/Gemini/Template
  - 🖥️ **Model**: The specific model being used (e.g., `gpt-oss:20b`)
  - 🎮 **Hardware**: Detected input device tier (if profiled)

**Backend Types**:
- **OLLAMA**: Local LLM running on your machine (private, no network)
- **GEMINI**: Google's cloud API (requires API key)
- **TEMPLATE**: Rule-based offline generation (always available)

### 2. Report Generation Panel

**Purpose**: Generate a comprehensive AI-powered lap debrief comparing your performance against a ghost lap.

**How to Use**:
1. Click the **"GENERATE LAP DEBRIEF"** button
2. Wait for the report to generate (30-60s for Ollama, 5-10s for Gemini, instant for Template)
3. The report will appear below with:
   - **Executive Summary**: One-paragraph overview
   - **Key Findings**: Bullet-point actionable insights
   - **Full Analysis**: Detailed markdown report with corner-by-corner breakdown

**What the Report Includes**:
- Corner-by-corner speed comparison
- Braking point analysis (early/late braking)
- Throttle application patterns
- Time loss/gain regions
- Actionable coaching tips scaled to your hardware

### 3. Report Display

**Features**:
- **Markdown Formatting**: Rich text with headers, tables, lists, and code blocks
- **Executive Summary**: Highlighted overview at the top
- **Key Findings**: Checkmark-bulleted list of top insights
- **Full Report**: Detailed analysis with professional race engineer tone
- **Generated By**: Shows which backend created the report

**Report Sections**:
1. **Executive Summary**: High-level performance overview
2. **Corner Analysis**: Turn-by-turn breakdown with speed deltas
3. **Braking Performance**: Brake point accuracy and recommendations
4. **Throttle Application**: Where to improve throttle usage
5. **Key Takeaways**: 3-5 actionable items for your next lap

### 4. How It Works Panel

**Purpose**: Educational panel explaining the 3-step intelligence pipeline.

**The Process**:
1. **ALIGNMENT**: Your lap and ghost lap are normalized onto a common distance grid using S-curve interpolation
2. **ANALYSIS**: Corner detection, delta computation, and coaching rules identify where time is lost or gained
3. **SYNTHESIS**: AI generates a natural language report with actionable insights tailored to your hardware

## Understanding the Reports

### Time Deltas

- **Positive (+)**: You are slower than the ghost
- **Negative (-)**: You are faster than the ghost
- **Example**: `+0.523s` means you were 523ms slower than the ghost lap

### Speed Deltas

- **Positive (+)**: You are faster at this point
- **Negative (-)**: You are slower at this point
- **Example**: `-5.2 km/h` means you were 5.2 km/h slower through that corner

### Corner Classifications

- **SLOW**: Tight hairpins (e.g., Monaco T1)
- **MEDIUM**: Standard corners (e.g., Silverstone Copse)
- **FAST**: High-speed sweepers (e.g., Spa Eau Rouge)

### Coaching Tip Severity

- **CRITICAL**: Major time loss (>100ms per occurrence)
- **HIGH**: Significant improvement opportunity (50-100ms)
- **MEDIUM**: Moderate gains available (20-50ms)
- **LOW**: Minor optimization (<20ms)

## Hardware Profiling

The intelligence layer automatically scales coaching expectations based on your input device:

### Hardware Tiers

1. **Keyboard** (Tier 1)
   - Binary inputs (0% or 100%)
   - Relaxed braking thresholds
   - Focus on racing line and timing

2. **Gamepad** (Tier 2)
   - Analog inputs with moderate precision
   - Standard coaching thresholds
   - Balance between precision and consistency

3. **Entry Wheel** (Tier 3)
   - Basic force feedback
   - Tighter thresholds for braking
   - Focus on smoothness

4. **Pro Wheel** (Tier 4)
   - High-end direct drive
   - Strictest thresholds
   - Professional-level expectations

**How It Works**:
- The system analyzes steering variance and frequency patterns
- Coaching tips are automatically adjusted to your hardware capabilities
- A keyboard player won't be told to "smooth out throttle application" like a wheel user would

## Backend Configuration

### Using Ollama (Recommended for Privacy)

**Advantages**:
- ✅ Completely private (no data leaves your machine)
- ✅ No API costs
- ✅ Works offline
- ✅ High-quality reports with 20B parameter model

**Setup**:
```bash
# Install Ollama
# Visit: https://ollama.ai

# Pull the model
ollama pull gpt-oss:20b

# Start the server
ollama serve
```

**Performance**:
- First request: 30-60s (model loading)
- Subsequent requests: 20-40s (generation)
- Requires: 16GB+ RAM, GPU recommended

### Using Gemini (Cloud Alternative)

**Advantages**:
- ✅ Fast generation (5-10s)
- ✅ No local hardware requirements
- ✅ Free tier available

**Setup**:
```bash
# Get API key from: https://ai.google.dev
export GEMINI_API_KEY="your-api-key-here"

# Restart the backend
python api/main.py
```

**Limitations**:
- Requires internet connection
- Data sent to Google servers
- API rate limits apply

### Template Mode (Offline Fallback)

**When It Activates**:
- Ollama not running
- Gemini API key not set
- LLM backends fail or timeout

**What You Get**:
- Rule-based report generation
- Instant results (<100ms)
- Structured data presentation
- No natural language synthesis

**Quality**:
- ✅ Accurate data and metrics
- ✅ All coaching tips included
- ❌ Less natural language flow
- ❌ No contextual insights

## Troubleshooting

### "Failed to connect to intelligence backend"

**Cause**: FastAPI backend is not running

**Solution**:
```bash
# From project root
python api/main.py
```

### "Report generation failed"

**Possible Causes**:
1. Ollama not running (if Ollama backend selected)
2. Invalid Gemini API key
3. Network timeout

**Solutions**:
1. Check Ollama: `curl http://localhost:11434/api/tags`
2. Verify API key: `echo $GEMINI_API_KEY`
3. Check backend logs for detailed error messages

### Report Takes Too Long

**If Using Ollama**:
- First request can take 60s+ (model loading)
- Subsequent requests are faster
- Consider using Gemini for faster results

**If Using Gemini**:
- Check internet connection
- Verify API rate limits not exceeded

### Report Quality Issues

**Template Mode Active**:
- Check if backend status shows "TEMPLATE"
- Set up Ollama or Gemini for AI-generated reports

**Ollama Model Not Found**:
- Verify model is pulled: `ollama list`
- Pull the model: `ollama pull gpt-oss:20b`

## Tips for Best Results

### 1. Complete Clean Laps
- Finish the full lap without crashes
- Avoid cutting corners (invalidates telemetry)
- Maintain consistent racing line

### 2. Choose Appropriate Ghost Laps
- Use professional drivers for aspirational comparison
- Use similar-skill drivers for realistic coaching
- Match track conditions (dry/wet)

### 3. Profile Your Hardware
- Upload a representative steering trace
- System will scale coaching to your capabilities
- Re-profile if you upgrade hardware

### 4. Review Reports Systematically
- Focus on **Key Findings** first
- Prioritize **CRITICAL** and **HIGH** severity tips
- Work on one corner at a time

### 5. Track Progress Over Time
- Generate reports for the same track/ghost combo
- Compare time deltas across sessions
- Measure improvement in specific corners
